# Library 项目技术参考

更新时间：2026-05-17

## 1. 架构概览

```text
用户手机 uni-app（微信小程序或 App）──HTTP/HTTPS──> Spring Boot API <── 电脑浏览器 Vue 管理端
                                      │
                                   MySQL
```

- 管理端：Vue 3 Web，电脑浏览器访问 `/admin/**`
- 用户端：uni-app（Vue3 + Vite + Pinia），可构建微信小程序或 App，读者手机使用
- 后端：统一 Spring Boot API，按角色和路径控制权限

## 2. 权限与路由

| 路径模式 | 允许角色 | 说明 |
|----------|----------|------|
| `/api/admin/**` | SUPER_ADMIN, ASSISTANT_ADMIN | 管理端接口 |
| `/api/user/**` | USER | 小程序用户端接口 |
| `/api/account/**` | USER | 账户中心接口 |
| `/api/auth/login` | 公开 | 小程序/App 登录（允许 role=1/0/2，拒绝游客） |
| `/api/admin/login` | 公开 | 管理端登录（兼容） |

## 3. 数据模型

### 业务主表

- `sys_user`：用户（含 nickname/email/avatar/role/status）
- `book_category`：图书分类
- `book_info`：图书主信息（书目级）
- `book_copy`：图书副本（单册级，copy_code 唯一）
- `borrow_record`：借阅记录（含 copy_id）
- `isbn_metadata`：本地 ISBN 元数据缓存
- `shelf_location`：库位
- `operation_log`：馆务操作日志

### 图片存储

- 用户头像：`sys_user.avatar`（URL 或 Base64）
- 图书封面：`book_info.cover_image`、`isbn_metadata.cover_image`（URL）
- 条码图片：后端动态生成，不落库不落盘

### 4.3 用户端（小程序）

| 接口 | 返回类型 | 说明 |
|------|----------|------|
| `GET /api/account/me` | SysUser | 当前用户资料（需 USER 角色） |
| `PUT /api/account/me/profile` | SysUser | 修改个人资料 |
| `GET /api/user/books` | Page\<BookInfo\> | 图书列表（支持 keyword/categoryId/availableOnly） |
| `GET /api/user/books/{id}` | BookInfo | 图书详情 |
| `GET /api/user/books/lookup` | BookInfo | 按 ISBN 查询 |
| `GET /api/user/books/copy-lookup` | CopyLookupView | 扫码副本查询（含 borrowable/returnable/message） |
| `POST /api/user/borrows` | BorrowActionResult | 创建借阅（含 dueTime/bookTitle/copyCode） |
| `GET /api/user/borrows/me` | Page\<BorrowRecordView\> | 我的借阅（批量查询，含书名/作者/ISBN/封面/副本码/到期状态） |
| `PUT /api/user/borrows/{id}/return` | ReturnActionResult | 按记录归还（含 returnTime/bookTitle/copyCode） |
| `PUT /api/user/borrows/return-by-copy-code` | ReturnActionResult | 扫码归还 |

| DTO | 字段 | 用途 |

| BorrowActionResult | recordId/bookId/bookTitle/copyId/copyCode/borrowTime/dueTime/status/message | 借阅结果 |
| ReturnActionResult | recordId/bookTitle/copyCode/returnTime/status/message | 归还结果 |

## 5. 小程序架构（uni-app）

### 目录

```text
miniprogram/                          # uni-app 工程（唯一用户端源码）
  public/
    tab/                              # tabBar 图标备份，不建议 pages.json 直接引用
  src/
    main.js                           # 入口，挂载 Pinia
    App.vue                           # onLaunch → auth.hydrate()
    manifest.json / pages.json
    config/env.js                     # 环境配置（devtools/lan/tunnel/prod）
    api/
      request.js                     # uni.request 统一封装（token/401+403节流/silent/timeout）
      auth.js / account.js / books.js / borrows.js
    stores/
      auth.js                         # Pinia: token/user/hydrate/login/logout/fetchMe
    utils/
      format.js                       # formatDateTime/formatDueState/statusText
      scan.js                         # uni.scanCode + showManualInput
      guard.js                        # requireLogin / requireAdmin / redirectToLogin / redirectToHome
    styles/
      tokens.scss                     # $mp-* 设计变量
      utilities.scss                  # .mp-page .mp-section-title 等
      components.scss                 # .mp-card .mp-button .mp-fab .mp-tag .mp-skeleton .mp-bottom-bar
    components/
      BookCard.vue / BorrowCard.vue / StatusTag.vue / ScanFab.vue / AppEmpty.vue
    pages/
      login/ books/ book-detail/ borrows/ profile/ bind/ admin
    static/tab/                       # tabBar 图标（uni-icons 生成的 64x64 本地 PNG，实际引用目录）
  dist/build/app/                     # App 构建产物（HBuilderX 真机运行）
  dist/build/mp-weixin/               # 微信小程序构建产物（开发者工具导入此目录）
```

### 技术栈

- Vue 3 + Vite + Pinia（与管理端一致）
- @dcloudio/uni-ui（easycom 自动注册）
- sass 预处理
- 构建命令：`npm run dev:mp-weixin` / `npm run build:mp-weixin` / `npx uni build -p app`

### 安全措施

- `/api/auth/login` 服务端允许 USER/SUPER_ADMIN/ASSISTANT_ADMIN 登录，拒绝 GUEST
- `RoleConstants.toRoleName(null)` 抛异常；JwtAuthenticationFilter 拒绝 null role token
- `/api/account/**` 需 USER 角色
- 统一 401/403 JSON 响应（JwtAuthenticationEntryPoint + JwtAccessDeniedHandler）
- 请求层 401/403 自动清理 token 并跳登录页（节流防重复）
- 所有 tab 页 onShow 检查 requireLogin()
- 登录页自动检测已有 token 并跳转

## 6. 后端安全与性能

| 措施 | 说明 |
|------|------|
| findByCopyCode 防重复崩溃 | `getOne(wrapper, false)` |
| CopyLookupView.message 不为 null | 可借状态返回"可借阅" |
| findCopyLookupView 找不到副本 | 返回 `Result.error("副本不存在")` |
| BorrowRecordView 批量查询 | batchGetUsers/Books/Copies，10 条从 31 次降到 4 次 |
| Collectors.toMap 防重复 key | 加 merge function `(a, b) -> a` |
| 归还操作日志 | 使用 copyCode 而非 copyId |
| availableOnly 服务端过滤 | BookInfoService.getPage 支持 currentStock > 0 条件 |

```bash
cd frontend && npm run build
# Nginx 托管 dist，/api/ 反向代理到 Spring Boot
```

### 小程序环境

| 环境 | env 值 | apiBaseUrl |
|------|--------|------------|
| 电脑开发者工具 | devtools | http://127.0.0.1:8080/api |
| 手机真机调试 | lan | http://你的电脑局域网IP:8080/api |
| HTTPS 内网穿透 | tunnel | https://穿透域名/api |
| 正式版 | prod | https://你的正式API域名/api |

> 微信开发者工具导入 `miniprogram/dist/build/mp-weixin`（或 `dist/dev/mp-weixin`），不是源码目录。

> HBuilderX App 真机运行使用 `miniprogram/dist/build/app` 或直接运行 uni-app 项目的 App 平台。

### 用户端真机调试要点

- 手机真机不能请求 `127.0.0.1`，因为该地址指向手机本机。
- 默认 `miniprogram/src/config/env.js` 使用 `const env = 'devtools'`，请求 `http://127.0.0.1:8080/api`。
- 手机真机调试时需改为 `env = 'lan'`，并把 `lan.apiBaseUrl` 替换为电脑局域网 IP。
- 手机浏览器访问 `http://电脑局域网IP:8080/api` 返回 `401` 时，说明网络链路已通；401 只是缺少登录 token。
- Windows 防火墙需要允许后端 8080 端口入站。

### 用户端 tabBar 规格

- tabBar 路径统一使用 `static/tab/...`，不要直接引用 `public/tab/...`。
- 图标源文件位于 `miniprogram/src/static/tab/`，当前为 `64x64` PNG。
- App 端显示尺寸由 `pages.json` 控制：`height: 72px`、`fontSize: 13px`、`iconWidth: 38px`、`spacing: 3px`。
- 微信小程序产物会自动剥离不适用的 App/H5 tabBar 尺寸字段。

## 8. 注意事项

- 国内图书对 OpenLibrary/Google Books 命中率不稳定，建议长期维护 `isbn_metadata`
- Google Books 公共配额可能 429，生产建议配置自有 API Key
- 头像当前存数据库 Base64，体积大时影响性能，建议后续迁移对象存储
- tabBar 图标已使用 uni-icons 生成 64x64 本地 PNG，App 端显示宽度为 38px
- `manifest.json` 的 mp-weixin.appid 为空，正式版需填入
- 小程序正式版必须使用 HTTPS 合法域名 + 微信公众平台配置 request 合法域名
- 生产部署必须设置 `LIBRARY_JWT_SECRET`，并按实际域名设置 `LIBRARY_CORS_ALLOWED_ORIGINS`
